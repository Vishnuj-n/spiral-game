# Integration & Data Injection

The Spiral Game frontend is designed to be completely decoupled from a traditional backend architecture. Instead of querying a backend API for questions, it relies on a **"push" architecture** where an external system (like your NPM PDF-parser package) processes data and pushes a JSON payload directly into the React app.

This document details how the injection works, how it's handled by the application, and how you can simulate it for testing.

---

## 🏗️ How it Works

The frontend `App` component actively looks for question data via two primary injection methods:

### 1. Injected via React Props (Library Usage)
If your NPM module bundles the app as an exported React component, developers can pass the pre-generated JSON payload via the `questionsData` prop.

```tsx
import { App as SpiralGame } from '@spiral-game/spiral-game';

function MyDashboard() {
  const generatedJson = [ ... ]; // Array of questions generated by PDF parser
  return <SpiralGame questionsData={generatedJson} />;
}
```

### 2. Injected via Global Window (Standalone Build)
If your NPM module builds a static HTML file to serve as a standalone viewer, you can inject the data as a serialized JSON string attached to the browser's `window` object *before* the React bundle loads.

```html
<script>
  // This is injected by the Node server before serving index.html
  window.__SPIRAL_DATA__ = [
    {
      "id": 1,
      "level": 1,
      "question": "What is the capital of France?",
      "options": ["London", "Paris", "Berlin"],
      "correctIndex": 1,
      "points": 100
    }
  ];
</script>
<!-- Vite App Bundle Loads Here -->
```

Whenever the `App` component detects a valid `questionsData` array from either of these sources upon mounting, it **bypasses the "Start Game" splash screen** and instantly transitions the user into the gameplay flow.

---

## 🧪 How to Simulate and Test

Because the logic is injected securely prior to runtime, you can simulate this behavior seamlessly without setting up complex mock APIs.

### Option A: Edit the Development `index.html` (Local Web Server)
When testing via `npx nx serve spiral-game`:

1. Open `apps/spiral-game/index.html`.
2. Above the script tag loading `/src/main.tsx`, inject a `<script>` tag redefining `window.__SPIRAL_DATA__` with a raw array:

```html
<script>
  window.__SPIRAL_DATA__ = [
    {
      "id": 99,
      "level": 1,
      "question": "Simulated PDF parsing output working?",
      "options": ["Yes", "No", "Maybe", "Error"],
      "correctIndex": 0,
      "hint": "Check the screen.",
      "points": 500
    }
  ];
</script>
```

3. Save the file.
4. Go to `http://localhost:4200/`. You will see the local Vite server instantly bypass the UI and render the fake database questions.

### Option B: Falling back to Default Demo
If `__SPIRAL_DATA__` is undefined, the application displays a simple "Start Demo Game" button that triggers a fetch request to `public/data.json` for fallback testing only. No backend server is queried. In production, your package should always provide data.
